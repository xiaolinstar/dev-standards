# H5 项目通用开发规范

> 决策见 [ADR-0010](adr/0010-h5-project-baseline.md)。本文是"怎么用"。
>
> 适用对象：未来所有移动端 H5 项目（包括 C 端推广页、内嵌微应用、微信分享网页及中后台管理端）。

## 1. 视口与尺寸适配规范

为了在不同屏幕密度与长宽比下呈现稳定排版，统一遵循以下适配标准：

### 1.1 Viewport 声明

入口 `index.html` 必须声明包含安全区适配及防缩放的视口配置：

```html
<meta
  name="viewport"
  content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover"
/>
```

- `user-scalable=no` 规避双击缩放。
- `viewport-fit=cover` 激活刘海屏/全面屏底部安全区延伸。

### 1.2 Viewport (vw) 适配

- **设计基准**：一律以 `375px` 宽度作为设计稿和开发像素基准（对应 iPhone SE/8/X 视口宽）。
- **转换插件**：使用 `postcss-px-to-viewport-8-plugin` 在编译时自动将开发中书写的 `px` 转换为 `vw`。
- **大屏沙盒兼容**：为保证在 PC 大屏上展示不拉伸变形，外层必须提供限宽容器，在大屏设备上将容器视口宽度固定在 `375px~480px` 之间并居中展示（参考 [App.vue 模版](h5-admin.md#1-pc-兼容沙盒布局appvue)）。

### 1.3 物理安全区适配

全面屏刘海（安全顶部）及底部 Home 指示条（安全底部）在 fixed 悬浮定位时极易与元素冲突。必须利用 CSS 环境变量进行避让：

```css
/* 顶部导航安全区避让 */
.header-safe-area {
  padding-top: constant(safe-area-inset-top);
  padding-top: env(safe-area-inset-top);
}

/* 底部操作条安全区避让 */
.footer-safe-area {
  padding-bottom: constant(safe-area-inset-bottom);
  padding-bottom: env(safe-area-inset-bottom);
}
```

### 1.4 高清屏 1px 细线

物理像素（dpr > 1）在直写 `border: 1px` 时在真机上会显得粗糙。使用 Tailwind 缩放或 CSS 伪元素 `transform: scale(0.5)` 处理 1px 细线：

```css
.hairline-bottom {
  position: relative;
}
.hairline-bottom::after {
  content: "";
  position: absolute;
  left: 0;
  bottom: 0;
  width: 100%;
  height: 1px;
  background-color: #cbd5e1; /* slate-200 */
  transform: scaleY(0.5);
  transform-origin: 0 100%;
}
```

## 2. 移动端 UI 组件与交互手感（UX）规范

### 2.1 按钮规范（Buttons）

- **触控面积**：移动端按钮和点击区域的最小物理尺寸不得低于 `44px * 44px`，避免误触或无法点击。
- **按压弹性微动效**：重要交互的按钮必须增加轻量级缩放或变暗反馈：

  ```html
  <button
    class="active:scale-[0.97] active:opacity-90 transition-all duration-100"
  >
    确认提交
  </button>
  ```

- **防重提交（Debounce）**：任何网络写操作按钮，点击后必须立即添加 `loading` 或 `disabled` 状态，由全局拦截器兜底或由组件属性控制。

### 2.2 输入框规范（Fields）

- 表单输入区域在移动端必须以圆角卡片包装展示（例如 `van-cell-group inset`）。
- 输入框的 `placeholder` 文本统一格式：`请输入[字段名称]`。
- 全局禁用原生软键盘自动联想，避免系统阻碍输入：

  ```html
  <input autocomplete="off" autocapitalize="off" spellcheck="false" />
  ```

### 2.3 下拉框与选择器规范（Dropdowns & Pickers）

> [!WARNING]
> **严禁在 H5 中直接使用原生 `<select>` 标签。**

#### 交互规范模式

一律采用 **只读 Field 触发框 + 底部弹窗选择器** 组合：

1. **触发框**：展示一个 `readonly` 的 `van-field`，配置 `is-link` 在右侧展现箭头。
2. **选择器**：点击触发框，从底部弹出 `van-popup`，并内嵌 `van-picker`（或 `van-date-picker`）。

#### 代码范式

```vue
<!-- 触发只读输入框 -->
<van-field
  v-model="selectedValueLabel"
  is-link
  readonly
  label="选项类别"
  placeholder="请选择"
  @click="showPicker = true"
/>

<!-- 底部弹出选择器 -->
<van-popup v-model:show="showPicker" position="bottom" round>
  <van-picker
    :columns="columns"
    title="请选择类别"
    @confirm="onConfirm"
    @cancel="showPicker = false"
  />
</van-popup>
```

### 2.4 大列表加载规范（List Pagination）

> [!WARNING]
> **严禁在移动端 H5 页面中直接渲染“第一页、下一页”的数字翻页按钮。**

- **兜底标语**：当所有数据加载完毕后，必须展现清晰的底部灰色提示语：`"没有更多了"` 或 `"已加载全部数据"`。

**标准大列表防重加载范式（Vue 3 + Vant 4）**：

```vue
<script setup lang="ts">
import { ref, onMounted } from "vue";
import { showToast } from "vant";

interface ListItem {
  id: number;
  title: string;
}

const list = ref<ListItem[]>([]);
const loading = ref(false);
const finished = ref(false);
let offset = 0;
const limit = 10;

const loadData = async (reset = false) => {
  // 1. 防重复加载保护卫语句
  if (loading.value && !reset) return;

  if (reset) {
    offset = 0;
    list.value = [];
    finished.value = false;
  }

  loading.value = true;
  try {
    const res = await api.getList({ limit, offset });
    list.value.push(...res.data);
    offset += limit;
    if (res.data.length < limit) {
      finished.value = true;
    }
  } catch {
    showToast("数据加载失败");
  } finally {
    loading.value = false;
  }
};

const onLoad = () => {
  // 2. 状态阻断卫语句
  if (finished.value) return;
  loadData();
};

const onRefresh = () => {
  loadData(true);
};

onMounted(() => {
  // 3. 显式首次加载
  loadData(true);
});
</script>

<template>
  <van-pull-refresh v-model="loading" @refresh="onRefresh">
    <van-list
      v-model:loading="loading"
      :finished="finished"
      :immediate-check="false"
      finished-text="没有更多了"
      @load="onLoad"
    >
      <div v-for="item in list" :key="item.id" class="card-item">
        {{ item.title }}
      </div>
    </van-list>
  </van-pull-refresh>
</template>
```

## 3. 移动端性能最佳实践

移动端设备性能及网络状态多变，必须对资源进行严格管控：

### 3.1 首屏加载 3 秒定律

- **路由按需加载**：除登录页和主看板外，所有次级页面和弹窗模块必须采用动态导入实现按需拆包（Code Splitting）：

  ```typescript
  const OperationsConfig = () => import("@/views/operations/index.vue");
  ```

- **三方库按需引入**：对于 Vant 4 等 UI 库，使用 `unplugin-vue-components` 自动加载按需组件和样式，严禁在 `main.ts` 中全量引入。

### 3.2 字体加载降级

中文字体动辄几兆大小，严禁直接在移动端首屏加载大型中文字体包。统一使用系统默认 fallback 栈：

```css
font-family:
  -apple-system, BlinkMacSystemFont, "Helvetica Neue", "PingFang SC",
  "Microsoft YaHei", sans-serif;
```

### 3.3 图片优化规范

- **首选格式**：所有本地和 CDN 上的装饰性图片、插图，优先输出 **WebP** 格式（体积平均减少 30%~50%）。
- **LCP（首屏大图）加速**：如果是首屏的 Banner 或产品英雄图，设置 `fetchpriority="high"` 告知浏览器最高优先级获取：

  ```html
  <img src="banner.webp" fetchpriority="high" alt="Hero" />
  ```

- **非首屏懒加载**：大列表中的缩略图或次屏图片，必须开启原生懒加载或组件库懒加载：

  ```html
  <img src="lazy-thumb.jpg" loading="lazy" alt="Thumb" />
  ```

## 4. 移动端WebView容器兼容性规范

### 4.1 iOS 软键盘遮挡与点击错位（关键修复）

- **痛点**：在 iOS 微信 WebView 等环境内，由于页面高度被弹出的输入法软键盘顶起，收起软键盘后，
  页面整体出现下方白底（高度未恢复），且按钮的物理点击区域发生偏移（点击无反应）。
- **解决方案**：在所有的文本输入框（`van-field`、`input`、`textarea`）的 `blur` 事件中，触发页面微滚动重置：

  ```typescript
  const handleInputBlur = () => {
    // 强制页面微滚动，让 iOS WebView 重新校准页面高度和物理点击区域
    window.scrollTo(
      0,
      Math.max(
        document.body.clientHeight,
        document.documentElement.clientHeight,
      ),
    );
  };
  ```

### 4.2 弹窗背景橡皮筋穿透（滚动冲突）

- **痛点**：在移动端，当弹起 Popup 或 Modal 覆盖层时，如果用户在弹窗上划动，底层的 body 依然会触发弹性回弹，造成体验极其恶劣的“双重橡皮筋滚动”。
- **解决方案**：在弹窗打开时，给 body 动态添加样式以暂时锁死底层页面滚动，弹窗关闭后移除：

  ```typescript
  watch(showPopup, (val) => {
    if (val) {
      document.body.style.overflow = "hidden";
    } else {
      document.body.style.overflow = "";
    }
  });
  ```

### 4.3 微信 WebView / 浏览器特定规范

- **单页重定向防抖**：在使用微信网页授权（OAuth2.0）换取 code 重定向时，必须清洗 URL 中的临时 code 参数，防止用户在分享当前链接或刷新时因 code 过期报 500 错误。
- **自定义分享卡片**：所有的前台/营销 H5 页面应统一接入微信 JS-SDK，并在页面加载时配置微信分享的标题、缩略图和描述：

  ```typescript
  wx.ready(() => {
    wx.updateAppMessageShareData({
      title: document.title,
      desc: "赶快来查看最新的打卡运营数据吧！",
      link: window.location.href.split("#")[0],
      imgUrl: "https://yourdomain.com/share-thumb.png",
    });
  });
  ```
