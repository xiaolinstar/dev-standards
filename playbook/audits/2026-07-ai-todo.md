| #     | 问题                                                                         | 严重度      | 修复建议                                                                                                                                                                    |

## 合规评分卡

### 1. 本地 Hooks 与 Git 规范

- [✅] `.husky/pre-commit` 存在，且安全集成了 gitleaks 与 lint-staged。
- [✅] `.husky/commit-msg` 存在，且绑定了 commitlint。
- [✅] `commitlint.config.cjs` 正常配置。
- [✅] 根 `package.json` 包含 `"prepare": "husky"` 自启动钩子。
- [✅] 根 `package.json` 包含 lint-staged 精准路径分流（且子项目路径相对化处理完美）。
- [✅] 最近 5 个 commit messages 均符合 Conventional Commits 规范。

### 2. CI/CD 流水线规范 (`ci.yml` / `cd.yml`)

- [✅] 包含阻塞式 `scan-secrets` (gitleaks) 阶段，且位于 Phase 0。
- [✅] 包含并行的 `scan-node` / `scan-miniapp` / `scan-api` 静态检查。
- [✅] 包含独立的 pytest 单元测试阶段。
- [✅] CD 流水线通过 `environment` 变量动态绑定 GitHub Environments，密钥分层极其清晰。

### 3. 环境配置与密钥管理 (L0 - L3)

- [✅] 包含完整的 `.env.*.example` 环境变量声明模板，且无真实敏感数据。
- [✅] 运行时密钥均只保存在 L3 运行环境（本地或服务器），未打包进 Docker 镜像层。

### 4. Kubernetes 部署文件规范

- [❌] **资源限额定义**：`api`/`worker`/`postgres` 容器的 resources limit/request 全部缺失。
- [❌] **哈希后缀配置**：禁用了配置生成器哈希，导致配置更新无法联动 Pod 滚动重启。
- [❌] **标签合规度**：仍然使用自定义简写标签，未采用 `app.kubernetes.io/*`。
- [⚠️] **探针完整性**：缺失 Startup Probe 启动探针，仅配置了 readiness/liveness。
